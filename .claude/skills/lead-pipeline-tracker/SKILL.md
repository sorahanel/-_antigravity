# 리드 파이프라인 트래커

## 기능

1. **리드 조회**: CSV에서 조건별 리드 필터링 (고객사, 단계, 유형, 담당자)
2. **파이프라인 요약**: 단계별 리드 수 및 예상 매출 집계 보고서
3. **액션 아이템**: 다음 액션이 필요한 리드 우선순위 정렬
4. **기한 알림**: 마감 임박 리드 자동 식별 및 경고

## CSV 스키마

| 컬럼명 | 설명 | 예시 값 |
|--------|------|---------|
| lead_id | 고유 식별자 | PUB-001, INF-003 |
| category | 구분 | 공공AX, AI인프라 |
| client | 고객사/과제명 | 삼성생명, 2026 AI행정혁신 |
| type | 유형 | 신규, 윈백-NCP, 윈백-AWS, 확대 |
| stage | 파이프라인 단계 | 발굴, 접촉, 제안, 협상, 클로징, 수주 |
| current_csp | 현재 사용 CSP | AWS, NCP, Azure, 없음 |
| expected_revenue | 예상 매출(만원) | 50000 |
| probability | 수주 확률(%) | 60 |
| next_action | 다음 액션 | PoC 제안서 작성 |
| due_date | 액션 기한 | 2025-07-15 |
| owner | 담당자 | 홍길동 |
| notes | 비고 | NCP 계약 만료 2025.09 |

## CSV 파일 위치

- 공공 AX: `data/public-ax-leads.csv`
- AI 인프라: `data/ai-infra-leads.csv`

## 출력 형식

### 파이프라인 요약 보고서

```
📊 [카테고리] 파이프라인 현황 (YYYY-MM-DD 기준)

| 단계 | 건수 | 예상 매출(만원) | 가중 매출(만원) |
|------|------|----------------|----------------|
| 발굴 | 3 | 150,000 | 15,000 |
| 접촉 | 2 | 80,000 | 24,000 |
| 제안 | 1 | 50,000 | 25,000 |
| 협상 | 1 | 30,000 | 21,000 |
| 합계 | 7 | 310,000 | 85,000 |

⚠️ 긴급 액션 필요:
- [lead_id] [client]: [next_action] (기한: [due_date])
```

가중 매출 = 예상 매출 × 수주 확률(%)

### 개별 리드 상세

```
🔎 [lead_id] [client]
━━━━━━━━━━━━━━━━━━━━━━━━
유형: [type] | 단계: [stage]
현재 CSP: [current_csp]
예상 매출: [expected_revenue]만원 (확률 [probability]%)
다음 액션: [next_action] (기한: [due_date])
담당자: [owner]
비고: [notes]
```

### 기한 임박 리드 알림

```
⏰ 기한 임박 리드 (7일 이내)
🔴 [오늘/내일] lead_id - client: next_action
🟡 [3일 이내] lead_id - client: next_action
🟢 [7일 이내] lead_id - client: next_action
```

## 사용 예제

**입력**: "이번 달 마감 임박한 리드 보여줘"
**처리**: CSV에서 due_date가 이번 달인 리드를 필터링, due_date 오름차순 정렬
**출력**:
```
⏰ 이번 달 액션 필요 리드 (2025-07 기준)

🔴 INF-002 삼성생명 [윈백-NCP] - 협상 단계
   → PoC 결과 보고서 제출 (기한: 07-10)

🟡 PUB-003 AI행정혁신 [신규] - 제안 단계
   → RFP 응답서 초안 완성 (기한: 07-20)

🟢 INF-004 버블챗 [윈백-AWS] - 접촉 단계
   → 비용 비교 자료 전달 (기한: 07-25)
```

**입력**: "전체 파이프라인 요약해줘"
**처리**: 양쪽 CSV 로드 후 카테고리별 집계
**출력**: 공공AX + AI인프라 파이프라인 요약 테이블 2개 + 전체 합계
